Introduction



The objective of this case study is to develop a predictive model for daily bike rentals in Chicago city. The dataset comprises various features, including weather conditions, temporal data, and operational days, which potentially influence bike rental patterns. The analysis is divided into the following sections:

  • Exploratory Data Analysis and Data Preprocessing.
  • Feature Engineering.
  • Candidate Model Development and Final Model Selection.


The aim is to uncover insights and build a model that accurately forecasts bike rental demand.

## Feature Engineering ##

# get hours and rush hours in time feature
time_list = df_full['DateHour'].tolist()
for i in range(len(time_list)):
    time_list[i] = time_list[i][11:13] 
df_full['hour'] = time_list
df_full['hour'] = df_full['hour'].astype(float)

# get months
time_list = df_full['DateHour'].tolist()
for i in range(len(time_list)):
    time_list[i] = time_list[i][5:7] 
df_full['month'] = time_list
df_full['month'] = df_full['month'].astype(float)

# Use temperature and wind speed to calculate the wind chill for different days as these can affect rental comfort
df_full['Wind_Chill']=(35.74 + 0.6215 * df_full['Temperature(F)'] - 35.75 * (df_full['Wind speed (mph)']** 0.16) + 0.4275* df_full['Temperature(F)'] * (df_full['Wind speed (mph)']** 0.16))

# get dummies
df_full = pd.get_dummies(df_full, columns=['Holiday', 'FunctioningDay'], drop_first=True, dtype=int)

  • 'hour': Bike rentals may vary throughout the day. Extracting hour from the 'DateHour' column allows us to capture daily rental patterns which identifying peak rental times.

Train Score:

The train score indicates the coefficient of determination (R-squared) for the training data.

Test Score:

The test score represents the R-squared value for the test data. It indicates how well your model generalizes to unseen data.

Train-Test Gap:

The train-test gap refers to the absolute difference between the train score and the test score.

OLS Linear Regression, Lasso Regression and Ridge Regression

There is not a huge difference between the scores of OLS Linear Regression, Lasso Regression and Ridge Regression in this case. They provide similar Train_Scores around 0.56 and Test_Scores around 0.53. A small Train_Test gap suggests that the models is not overfitting excessively, as it performs relatively similarly on both the training and test datasets.

Elastic Net Regression

The scores for the Elastic Net Regression seem unusually large and negative.

K-Nearest Neighbors

The K-Nearest Neighbors model seems to have a good performance on the training data, with an Train_Score of 0.6039. However, its performance drops on the test data, with a Test_Score of 0.3316. The relatively large train-test gap of 0.2723 suggests that the model might be overfitting to the training data.

Decision Tree Regressor

We got a high score 1.0 on our training data and less score on test data. The model overfitted.

    # Access the trained model directly as the last step of the pipeline
    trained_model = pipeline.steps[-1][1]  # steps is a list of tuples, -1 accesses the last step, 1 accesses the model

    # Feature importances or coefficients
    if hasattr(trained_model, 'feature_importances_'):
        importances = trained_model.feature_importances_
    elif hasattr(trained_model, 'coef_'):
        importances = np.abs(trained_model.coef_[0])

    # Get the top two features
    top_two_indices = np.argsort(importances)[-2:]
    top_two_features = [x_train.columns[i] for i in top_two_indices]
